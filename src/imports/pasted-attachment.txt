FIELD OFFICER MODULE CORRECTIONS

QR SCANNER PAGE WORKFLOW

The QR Scanner page is used to verify vendor payment.

When the officer scans a vendor QR code the system must display:

Vendor Name
Vendor ID
Stall Type
Market Name
Payment Status

If the payment status is **Paid**, show:

Payment Status: Paid (green indicator)

If the payment status is **Not Paid**, show:

Payment Status: Not Paid (red indicator)

Display a button:

Collect Fee

When the officer clicks **Collect Fee**, navigate to the **Fee Collection page**.

---

COLLECTION PAGE RENAME

Rename the page currently called **Collections** to:

Fee Collection

---

FEE COLLECTION PAGE PURPOSE

The Fee Collection page is used only for collecting market stall fees.

It must support two cases:

1. Vendor coming from QR Scanner
2. New vendor manual entry

---

CASE 1: VENDOR FROM QR SCAN

If the officer came from the QR Scanner page, automatically display:

Vendor Name
Vendor ID
Stall Type
Market Name

Display the fee amount based on stall type (₹20–₹100).

Add a button:

Collect Payment

After payment show options:

Generate Digital Receipt
Print Receipt

---

CASE 2: NEW VENDOR MANUAL ENTRY

If the vendor is new and has no QR code, allow manual vendor entry.

Display input fields:

Vendor Name
Mobile Number
Stall Type (Dropdown)

Stall Type options:

Vegetables
Fruits
Greens
Fish
Meat
Poultry
Groceries
Grains
Flowers
Pots
Iron Tools
Clothes
Toys
Livestock
Others

System automatically calculates the stall fee.

Officer presses:

Collect Payment

After payment the system generates:

Vendor ID
Digital Receipt

PANCHAYAT OFFICER MODULE CORRECTIONS

1. MARKET MANAGEMENT TEXT CORRECTION

In the Panchayat Officer → Market Management page, the title currently says:

"Weekly markets across Tamil Nadu"

Change it to:

"Weekly Markets in This Panchayat"
or
"Weekly Markets Under This Panchayat"

The markets displayed must represent only the officer’s local panchayat area, not the whole state.

Stall Type
Registered Market
